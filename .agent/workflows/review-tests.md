# Review Tests

Examine tests added in the PR.

1. **Check for Overengineering**
   - Ensure tests verify functionality without unnecessary complexity.

2. **Check Parameterization**
   - Reduce LOC by parametrizing similar test cases.

3. **Check DRY**
   - Ensure tests follow Don't Repeat Yourself principle.

4. **Verify Usefulness**
   - Confirm tests actually provide value and coverage.
